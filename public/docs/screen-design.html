1. 커머스 홈

앱의 진입점. 배너, 카테고리, 추천 상품을 한 화면에 노출한다.

구성요소

TitleBar (+ HomeCartIcon) BannerCarousel CategoryGrid (3×2) ProductCard Grid (2열) BottomTabBar (4탭)

데이터

항목소스비고
배너banners 테이블is_active=1, sort_order 순
카테고리categories 테이블6종, 이모지 아이콘
추천 상품products + skus품절 아닌 상위 6개

인터랙션

  • 배너 좌우 스와이프 / 인디케이터 dot 탭
  • 카테고리 탭 → /products?category={slug}
  • 상품 카드 탭 → /products/{id}
  • 하단 탭바 네비게이션

상태

기본

배너 + 카테고리 + 추천 상품 표시

빈 상태

해당 없음 (시드 데이터 보장)

1.5 단백질 스캐너

3문항 설문으로 사용자에게 맞는 단백질 쉐이크 Top 3를 추천하는 인터랙티브 화면. 하단 탭바 2번째 탭(/scanner). 추천 데이터는 화면 전용 정적 데이터이며 DB와 무관하다.

구성요소

진행 헤더 (뒤로 + NN/03 카운터) QuestionCard (단일선택 옵션 버튼) ResultCard ×3 (Badge No.1~3) Button (처음으로)

데이터

항목소스비고
설문 문항scanner/data.ts QUESTIONS3문항(목적/부담/식감), 정적
추천 상품scanner/data.ts PRODUCTS단백질 쉐이크 8종, 정적 (DB 무관)
추천 로직scanner/recommend.ts태그 매칭 + 가중치 → Top 3

플로우

  • step 0~2: 질문 표시. 옵션 탭 시 답변 저장 후 다음 단계로 자동 진행
  • step 3: 결과. 선택 답변의 태그를 가중치(목적×3, 부담×2, 식감×1)로 합산해 매칭 점수 상위 3개 노출
  • 각 결과 카드: 순위 뱃지(No.1은 강조), 상품명·맛/용량(mono), 한 줄 소개, 추천 이유 칩
  • 뒤로: step 0에서는 /home으로, 이후 단계는 이전 질문으로
  • "처음으로" → 답변 초기화 후 step 0

상태

질문

NN/03 카운터 + 옵션 버튼, 선택 시 강조

결과

RESULT 라벨 + 추천 3종 + 처음으로

3. 상품 상세

상품 정보 + 옵션 선택 + 구매 액션이 있는 핵심 화면.

구성요소

BackHeader ProductImage OptionSelector QuantitySelector BottomActionBar

데이터

항목API비고
상품 상세GET /api/products/{id}옵션 그룹 + SKU 매트릭스

OptionSelector 로직

  • 옵션 없음: 단일 SKU, 가격과 수량만 표시
  • 단일 옵션: 1개 그룹 선택 → SKU 확정
  • 조합형 옵션: 2개 이상 그룹 모두 선택 → SKU 확정
  • 품절 비활성화: 특정 옵션값을 포함하는 모든 SKU가 stock=0이면 해당 버튼 비활성 + 취소선 + "(품절)"
  • 재고 임박: stock ≤ 5일 때 amber 뱃지 "N개 남음"
  • 수량: 1 ~ stock, SKU 확정 후 표시

인터랙션

  • 옵션 버튼 탭 → 선택 상태 토글 (비활성 불가)
  • "장바구니" 버튼 → POST /api/cart + 토스트
  • "구매" 버튼 → POST /api/cart + /checkout 이동
  • 옵션 미선택 시 하단 버튼 비활성

상태

기본

상품 정보 + 옵션 선택 UI

전체 품절

이미지 오버레이 + 구매 버튼 비활성

로딩

스피너 표시

404

3.5 상품 리뷰

상품 상세 화면 내 리뷰 조회·작성 기능. 리스트와 작성 폼은 BottomSheet로 노출된다.

구성요소

ReviewSummary (평균 별점 + 개수) ReviewList (BottomSheet) ReviewForm (별점 + 본문 + 사진) PhotoUploader (최대 5장)

데이터

항목API비고
리뷰 조회GET /api/products/{id}/reviews상품별 리뷰 목록
리뷰 작성POST /api/products/{id}/reviews별점 1~5, 본문, 사진 URL 배열
사진 업로드POST /api/upload이미지 전용, 최대 5MB → public/uploads/reviews/

인터랙션

  • 리뷰 요약 탭 → 리뷰 리스트 BottomSheet 열기
  • "리뷰 작성" → 작성 폼 BottomSheet (별점 선택, 본문 입력, 사진 첨부 최대 5장)
  • 사진 선택 → /api/upload 업로드 후 썸네일 미리보기
  • 제출 → /api/products/{id}/reviews, 토스트 후 리스트 갱신

제약

  • 세션당 상품별 1건 (UNIQUE idx_reviews_session_product) — 중복 작성 시 거부
  • 별점 필수 (1~5), 본문·사진은 선택

구성요소

데이터

항목API비고
장바구니GET /api/cartSKU 상세 + 재고 포함
수량 변경PATCH /api/cart/{id}1 ≤ qty ≤ stock
항목 삭제DELETE /api/cart/{id}개별 / 선택 삭제

인터랙션

  • 전체 선택 / 개별 선택 체크박스
  • 수량 +/- 버튼 (재고 범위 제한)
  • 개별 삭제 (✕) / 선택 삭제
  • "주문하기" → /checkout 이동

가격 분해

항목계산
상품금액Σ(price × quantity)
배송비상품금액 ≥ 30,000원 → 무료, 미만 → 3,000원
총 결제금액상품금액 + 배송비

상태

기본

장바구니 항목 + 가격 분해

빈 상태

"장바구니가 비어있습니다" + 상품 링크

5. 체크아웃

구성요소

BackHeader OrderItemsSummary ShippingForm (4개 필드) PriceBreakdown WalletBalance Modal (에러)

데이터

항목API비고
장바구니GET /api/cart주문 요약용

  • 성공 → /order-complete 이동

결제 중

8. 주문 상세

개별 주문의 상세 정보 + 취소 / 반품 / 교환 기능.

구성요소

StatusBadge OrderItems DeliveryInfo PriceBreakdown CancelButton + ConfirmModal Return/ExchangeButton + ReasonSheet

상태별 액션

액션노출 조건API처리
취소결제완료 / 배송준비POST /api/orders/{id}/cancel재고 원복 + 전액 환불 → 취소완료
반품배송완료POST /api/orders/{id}/return재고 원복 + 전액 환불 → 반품완료
교환배송완료POST /api/orders/{id}/exchange재고·지갑 변동 없음 → 교환완료

인터랙션

  • 취소 → 확인 모달 → 원자적 처리(재고 원복 + 잔액 환급 + 상태 변경)
  • 반품 → 사유 선택 BottomSheet(단순 변심 / 상품 불량 / 상품이 설명과 다름 / 배송 중 파손 / 기타) + 메모
  • 교환 → 사유 선택 BottomSheet(상품 불량 / 오배송 / 상품이 설명과 다름 / 배송 중 파손 / 기타) + 메모
  • 제출 시 토스트("반품이 완료되었습니다." / "교환이 완료되었습니다.") 후 상태 갱신
