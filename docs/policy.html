1. 상품/SKU 정책

상품 구조

상품(Product)은 카테고리에 속하며, 하나 이상의 SKU를 가진다. 옵션이 있는 상품은 옵션 그룹(OptionGroup)과 옵션 값(OptionValue)으로 구성된다.

옵션 유형

유형옵션 그룹 수SKU 수예시
옵션 없음01비타민D3
단일 옵션12~3맛: 오렌지/딸기
조합형 옵션24 (2×2)맛×용량

SKU 코드 규칙

형식: {카테고리}-{상품약어}-{옵션약어}. 예: VIT-MV-OR30 (비타민-멀티비타민-오렌지30정)

카테고리

6종: 비타민, 프로바이오틱스, 오메가3, 미네랄, 콜라겐, 어린이

상품 품절 판정

상품의 모든 활성 SKU의 stock이 0이면 해당 상품은 "품절"로 표시한다.

2. 가격 정책

가격 단위

모든 가격은 KRW 정수(INTEGER)로 저장한다. 소수점 없음.

가격 표시

  • 리스트: 가격 범위 (min_price ~ max_price) 또는 단일 가격
  • 상세: SKU 확정 후 해당 SKU 가격
  • 포맷: toLocaleString('ko-KR') + '원'

가격 변동

프로토타입에서는 가격 변동 없음. 주문 시 현재 SKU 가격을 주문 항목에 스냅샷 저장한다.

3. 배송 정책

항목값
기본 배송비3,000원
무료배송 기준상품금액 30,000원 이상
배송비는 장바구니와 체크아웃에서 실시간 계산된다. 상품금액(product_total) 기준이며, 할인 후 금액이 아닌 합산 금액으로 판단한다.

4. 장바구니 정책

식별

세션(쿠키) 기반. 동일 세션의 동일 SKU는 UNIQUE(session_id, sku_id)로 중복 방지한다.

수량 규칙

  • 최소 수량: 1개
  • 최대 수량: 해당 SKU의 현재 재고(stock)
  • 동일 SKU 재담기 시 수량 합산 (재고 이하로 cap)

품절 SKU 담기

stock=0인 SKU는 장바구니에 담을 수 없다. API에서 거부.

재고 변동

장바구니에 담겨있는 상태에서 재고가 감소할 수 있다. 최종 검증은 체크아웃 시 수행.

5. 결제 정책

결제 수단

가상 지갑 단일 결제. 실제 PG 연동 없음.

검증 순서

  1. 장바구니 항목 + 현재 SKU 데이터 조회
  2. 재고 재확인: 각 SKU stock ≥ 주문 수량
  3. 가격 계산: 현재 SKU 가격으로 합계 + 배송비
  4. 잔액 확인: 지갑 잔액 ≥ 총 결제금액
  5. 원자적 처리 (하단 참조)

원자적 트랜잭션

better-sqlite3의 .transaction()을 활용한다. 모든 단계가 동기적으로 실행되며, 예외 발생 시 자동 ROLLBACK된다.
잔액 차감 → 재고 차감 → 주문 생성 → 주문 항목 생성 → 지갑 거래 기록 → 장바구니 비우기

재고 차감 이중 안전장치

UPDATE skus SET stock = stock - ? WHERE id = ? AND stock >= ? — WHERE 조건으로 race condition 방지.

실패 처리

실패 사유UI 처리
재고 부족모달: "상품명" 재고 부족 안내
잔액 부족모달: 잔액/결제금액 표시
장바구니 비어있음/cart로 리다이렉트

6. 주문 정책

주문번호 생성

형식: ORD-{YYMMDD}-{랜덤6자}. 예: ORD-260614-A3B7K2

상태 플로우

결제완료 → 배송준비 → 배송중 → 배송완료
결제완료 → 취소완료

주문 취소 조건

규칙상세
취소 가능 상태결제완료, 배송준비만 가능
취소 범위전체 취소만 (부분 취소 없음)
환불결제 금액 전액 가상 지갑 환급
재고주문 항목의 수량만큼 재고 원복

취소 원자적 처리

재고 원복 → 잔액 환급 → 환불 거래 기록 → 상태 → 취소완료

주문 항목 스냅샷

주문 생성 시 product_name, option_summary, unit_price를 order_items에 복사 저장. 이후 상품 변경과 무관하게 원래 주문 정보를 보존.

7. 지갑 정책

초기 부여

  • 세션 최초 접근 시 자동 생성
  • 초기 잔액: 100,000원
  • "가상 지갑 초기 부여" 거래 기록 생성

거래 유형

유형발생 시점설명
적립지갑 생성초기 부여금
사용주문 결제총 결제금액 차감
환불주문 취소결제 금액 전액 환급

잔액 일관성

모든 잔액 변경은 트랜잭션 내에서 수행. balance_after는 거래 후 잔액을 기록하여 추적 가능.

세션 격리

UNIQUE(session_id) 제약. 세션별 독립 지갑.

8. 재고 정책

재고 단위

SKU 단위 관리. skus.stock (INTEGER, ≥ 0)

재고 분포 (시드 데이터)

범위설명해당 SKU
20~100일반 재고대부분
0품절2~3개 SKU (포도60정, 김치유산균3개월, 콜라겐 전체)
1~5재고 임박2~3개 SKU (포도30정, 비타민C180정, 콜라겐젤리 블루베리)

재고 임박 표시

stock ≤ 5 → amber 뱃지 "N개 남음" 표시 (상세 페이지 옵션 선택 후)

품절 표시

  • 상품 카드: 전체 SKU 품절 시 이미지 위 품절 뱃지
  • 옵션 버튼: 해당 옵션값의 모든 SKU 품절 시 비활성 + 취소선 + "(품절)"
  • 구매 버튼: 선택된 SKU 품절 시 비활성

재고 차감 시점

결제 완료 시점(체크아웃 트랜잭션)에만 차감. 장바구니 담기 시에는 차감하지 않음.

재고 원복

주문 취소 시 해당 주문 항목의 수량만큼 원복.
